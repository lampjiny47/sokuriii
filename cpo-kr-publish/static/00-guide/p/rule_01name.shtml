Nameing 규칙

목적

  1. 이름만 보고 무엇을 하는 지 알 수 있어야 한다.
  2. 이름만 보고 어디에서 사용하는 지 알 수 있어야 한다.
  3. Nameing 규칙을 지킴으로 소스의 재사용성을 높인다.

기본 규칙

  1. 영문, 숫자, 언더바(_)로 작성한다.
  2. selector, images는 대문자, 숫자로 시작할 수 없다.
  3. 언더바(_)는 2개 이상의 단어조합에 사용하되, 단어와 숫자의 조합일 경우 생략한다.
  4. 언더바(_)의 사용은 1개로 제한한다.
  5. 단어에 숫자조합을 하는 경우 0을 붙여 2자리로 표현한다.(예 : 01)
  6. selelctor는 컴퍼넌트별 약어로 시작한다.(Component 약어(표-1) 참조)
  7. images는 영역별 Sprite 약어 사용한다.(Sprite 영역별 이미지(표-3) 참조)
  8. 사이트 전체 큰 프레임을 형성하는 Layout은 ID로 정의하며 'CUV'가 들어가도록 네이밍한다.(레이아웃 참조)
  9. jQuery로 컨트롤 되는 기능은 별도 selector를 선언하며, jQ_ 를 붙인다.
  10. HTML 파일명은 스토리보드 화면ID로 작성하며, 확장자는 SSI(Server Side Include)를 위해 .shtml로 한다.
  11. 하나의 id는 문서에서 한번만 사용한다.
  12. 레이아웃 공통어(표-1) 및 기능상의 이유가 없다면 id의 사용은 자제한다.
  13. 레이아웃 id의 네이밍은 조합하지 않고, 디자인 속성을 추가/변경하려면 class를 사용한다.

selelctor Naming

  1. 기본 Component의 시작은 아래 'Component 약어(표-1)'에 명시된 약어로 시작한다.
  2. Component는 해당 Component가 갖는 디자인 요소만 style 처리한다. (레이아웃은 해당 컴포넌트를 감싸고 있는 요소에 넣기)
  3. 특정 영역에서만 사용되는 Component는 약어 앞에 특정영역 명을 넣을 수 있다. (예 : search_ipt)
  4. reset 스타일을 제외하고 selector 없이 html 태그에 스타일을 넣을 수 없다.
  5. inner, box, tit, txt 등 간략한 약어로 구성된 selector는 부모 selector에 상속 시켜 사용한다.
  6. 전체선택자(*)는 box-sizing:border-box를 제외하고 사용하지 않는다.
  7. 인접 형제선택자(Adjacent Sibling Combinator), 일반 형제 선택자(General Sibling Combinator)는 사용하지 않는다. (예 : li + li, .main_box ~ div)

SASS Naming(function & mixin)

  1. SASS문에서 FUNCTION와 MIXIN 의 기본 시작은 대분자로 표현한다.
  2. 약어를 사용하지 않는다.
  3. 약어를 사용하지 않는다.

image Naming

  1. 영문, 숫자, 언더바(_)로 작성한다.
  2. 이미지 공통어(표N-2)에 근거한다.
  3. 같은 분류의 이미지가 두 개 이상일 경우 숫자를 추가해 구분한다.
  4. Sprite 처리 이미지는 영역별로 묶어서 관리하며, 이름 앞에 영역별 약어를 사용한다.(Sprite 영역별 이미지(표-3))
  5. 속도를 고려하여 sprite 이미지 사이즈(600x600)를 용량을 제한한다.
  6. 컨텐츠 이미지는 폴더로 1차 구분한 후 컨텐츠에 맞는 영어명을 사용한다.
